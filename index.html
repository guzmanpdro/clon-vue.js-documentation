Introducción

¿Qué es Vue.js?

Vue (pronunciado /vjuː/, como view) es un framework progresivo para la construcción de interfaces de usuario. A diferencia de otros frameworks monolíticos, Vue está diseñado desde el principio para ser adoptado de forma incremental. El núcleo de la biblioteca se centra en la capa de vista solamente, y es fácil de recoger e integrar con otras bibliotecas o proyectos existentes. Por otro lado, Vue también es perfectamente capaz de impulsar sofisticadas aplicaciones de una sola página cuando se utiliza en combinación con herramientas modernas y bibliotecas de apoyo.

Empezando

La forma más fácil de probar Vue.js es utilizando el ejemplo de Hello World. Siéntete libre de abrirlo en otra pestaña y seguir a lo largo de algunos ejemplos básicos. O bien, puedes crear un archivo index.html e incluir Vue con:

                
<!-- development version, includes helpful console warnings -->
<script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>

or:

                
<!-- production version, optimized for size and speed -->
<script src="https://cdn.jsdelivr.net/npm/vue@2"></script>

La página de instalación proporciona más opciones de instalación de Vue. Nota: No recomendamos que los principiantes comiencen con vue-cli, especialmente si no están familiarizados con las herramientas de construcción basadas en Node.js.

Si prefieres algo más interactivo, también puedes echar un vistazo a esta serie de tutoriales en Scrimba, que te ofrece una mezcla de screencast y código de juego que puedes pausar y jugar en cualquier momento.

Renderizado declarativo

At the core of Vue.js is a system that enables us to declaratively render data to the DOM using straightforward template syntax:

                
<div id="app">
    {{ message }}
</div>
                
            
                
var app = new Vue({
    el: '#app',
    data: {
        message: 'Hello Vue!'
    }
})
{{message}}

¡Ya hemos creado nuestra primera aplicación Vue! Esto se parece bastante a renderizar una plantilla de cadenas, pero Vue ha hecho mucho trabajo bajo el capó. Los datos y el DOM están ahora vinculados, y todo es ahora reactivo. ¿Cómo lo sabemos? Abre la consola JavaScript de tu navegador (ahora mismo, en esta página) y establece app.message con un valor diferente. Deberías ver que el ejemplo renderizado de arriba se actualiza en consecuencia.

Observe que ya no tenemos que interactuar con el HTML directamente. Una aplicación Vue se adjunta a un solo elemento del DOM (#app en nuestro caso) y luego lo controla completamente. El HTML es nuestro punto de entrada, pero todo lo demás ocurre dentro de la instancia Vue recién creada.

Además de la interpolación de texto, también podemos vincular los atributos de los elementos de esta manera:

                
<div id="app-2">
    <span v-bind:title="message">
        Hover your mouse over me for a few seconds
        to see my dynamically bound title!
    </span>
</div>
                
            
                
var app2 = new Vue({
    el: '#app2',
    data: {
        message: 'You loaded this page on ' + new Date().toLocaleString()
    }
})
Hover your mouse over me for a few second to see my dynamically bound title!

Aquí nos encontramos con algo nuevo. El atributo v-bind que estás viendo se llama directiva. Las directivas llevan el prefijo v- para indicar que son atributos especiales proporcionados por Vue, y como habrás adivinado, aplican un comportamiento reactivo especial al DOM renderizado. Aquí, básicamente está diciendo "mantén el atributo title de este elemento actualizado con la propiedad message en la instancia de Vue".

Si vuelves a abrir tu consola de JavaScript e introduces app2.message = 'algún mensaje nuevo', verás de nuevo que el HTML vinculado -en este caso el atributo title- se ha actualizado.

Condicionales y bucles

También es fácil alternar la presencia de un elemento:

                
<div id="app-3">
    <span v-if="seen">Now you see me</span>
</div>
                
            
                
var app3 = new Vue({
    el: '#app-3',
    data: {
        seen: true
    }
})
Now you see me

Sigue adelante e introduce app3.seen = false en la consola. Deberías ver que el mensaje desaparece.

Este ejemplo demuestra que podemos enlazar datos no sólo con texto y atributos, sino también con la estructura del DOM. Además, Vue también proporciona un potente sistema de efectos de transición que puede aplicar automáticamente efectos de transición cuando los elementos son insertados/actualizados/eliminados por Vue.

Hay bastantes otras directivas, cada una con su propia funcionalidad especial. Por ejemplo, la directiva v-for puede utilizarse para mostrar una lista de elementos utilizando los datos de un Array: 

                
<div id="app-4">
    <ol>
        <li v-for="todo in todos">
            {{ todo.text }}
        </li>
    </ol>
</div>
                
            
                
var app4 = new Vue({
    el: '#app-4',
    data: {
        todos: [
            { text: 'Learn JavaScript' },
            { text: 'Learn Vue' },
            { text: 'Build something awesome' }
        ]
    }
})
  1. {{ todo.text }}

En la consola, introduce app4.todos.push({ text: 'New item' }). Deberías ver un nuevo elemento añadido a la lista.

Manejo de las entradas del usuario

Para permitir que los usuarios interactúen con tu aplicación, podemos utilizar la directiva v-on para adjuntar escuchadores de eventos que invoquen métodos en nuestras instancias de Vue.

                
<div id="app-5">
    <p>{{ message }}</p>
    <button v-on:click="reverseMessage">
        Reverse message
    </button>
</div>
                
            
                
var app5 = new Vue({
    el: '#app-5',
    data: {
        message: 'Hello Vue.js'
    },
    methods: {
        reverseMessage: function() {
            this.message = this.message.split('').reverse().join('')
        }
    }
})

{{ message }}

Ten en cuenta que en este método actualizamos el estado de nuestra aplicación sin tocar el DOM - todas las manipulaciones del DOM son manejadas por Vue, y el código que escribes se centra en la lógica subyacente.

Vue también proporciona la directiva v-model que hace que la vinculación bidireccional entre la entrada del formulario y el estado de la aplicación sea muy fácil:

                
<div id="app-6">
    <p>{{ message }}</p>
    <input v-model="message">
</div>
                
            
                
var app6 = new Vue({
    el: '#app'6,
    data: {
        message: 'Hello Vue!'
    }
})

{{ message }}

Componiendo con componentes

El sistema de componentes es otro concepto importante en Vue, porque es una abstracción que nos permite construir aplicaciones a gran escala compuestas por componentes pequeños, autocontenidos y a menudo reutilizables. Si lo pensamos bien, casi cualquier tipo de interfaz de aplicación puede abstraerse en un árbol de componentes:

En Vue, un componente es esencialmente una instancia de Vue con opciones predefinidas. Registrar un componente en Vue es sencillo:

                
//Define un nuevo componente llamado todo-item
Vue.component('todo-item', {
    template: '<li>This is a todo</li>'
})

var app = new Vue(...)

Ahora puede componerlo en la plantilla de otro componente:

                
<ol>
    <!--Crear una instancia del componente todo-item -->
    <todo-item></todo-item>
</ol>

Pero esto haría el mismo texto para cada cosa, lo que no es súper interesante. Deberíamos ser capaces de pasar datos del ámbito padre a los componentes hijos. Modifiquemos la definicion del componente para que acepte una prop:

                
Vue.component('todo-item', { 
    // El componente todo-item ahora acepta una "prop",
    // que es como un atributo personalizado.
    // Esta prop se llama todo */
    
    props: ['todo'],
    template: '<li>{{ todo.text }}</div>'
})

Ahora podemos pasar el todo a cada componente repetido usando v-bind:

                
<div id="app-7">
    <ol>
        <!--
            Ahora proporcionamos a cada todo-item el objecto todo que representa,
            para que su contenido pueda ser dinámico. También tenemos que proporcionar
            a cada componente una "clave", que se explicará más adelante.
        -->
        <todo-item
            v-for="item in groceryList"
            v-bind:todo="item"
            v-bind:key="item.id"
        ></todo-item>
    </ol>
</div>
                
            
                
Vue.component('todo-item', {
    props: ['todo'],
    template: '<li>{{ todo.text  }}</li>'
})

var app7 = new Vue({
    el: '#app-7',
    data: {
        groceryList: [
            { id: 0, text: 'Vegetables' },
            { id: 1, text: 'Cheese' },
            { id: 2, text: 'Whatever else humans are supposed to eat' }
        ]
    }
})

Este es un ejemplo artificial, pero hemos conseguido separar nuestra aplicación en dos unidades más pequeñas, y el hijo está razonablemente bien desacoplado del padre a través de la interfaz props. Ahora podemos mejorar nuestro componente <todo-item> con una plantilla y una lógica más complejas sin afectar a la aplicación principal.

En una aplicación grande, es necesario dividir toda la app en componentes para que el desarrollo sea manejable. Hablaremos mucho más sobre los componentes más adelante en la guía, pero aquí hay un ejemplo (imaginario) de cómo podría ser la plantilla de una app con componentes:

                
<div id="app">
    <app-nav></app-nav>
    <app-view>
        <app-sidebar></app-sidebar>
        <app-content></app-content>
    </app-view>
</div>
